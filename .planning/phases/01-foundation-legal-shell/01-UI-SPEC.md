# Phase 1 — UI Design Contract

> Visual and interaction contract for Phase 1: Foundation & Legal Shell.
> Generated by gsd-ui-researcher. Verified by gsd-ui-checker.
>
> **Scope:** BootScene (loading progress bar), PreloadScene (branded loading bar),
> MainMenuScene ("Tap to Start" gate + legal footer links), SettingsScene stub
> (music/SFX toggles), static HTML legal pages (impressum.html, datenschutz.html).
> All game UI is rendered to a Phaser 4 canvas; only legal pages are plain HTML.

---

## Design System

| Property | Value | Source |
|----------|-------|--------|
| Tool | none | RESEARCH.md — Phaser 4 canvas, not a React component library |
| Preset | not applicable | no shadcn; stack is Phaser 4 + Vite + TypeScript |
| Component library | Phaser 4 GameObjects (Text, Rectangle, Graphics, Image) | RESEARCH.md Standard Stack |
| Icon library | none — pixel-art sprites only; UI icons hand-drawn in 16x16 tileset | default (pixel-art constraint, PROJECT.md) |
| Font | Press Start 2P (Google Fonts) — 8px bitmap fallback via Phaser BitmapFont | default — canonical retro game font; matches 16-bit pixel-art style |
| CSS scope | Only for the HTML shell and static legal pages; Phaser canvas styled via `image-rendering: pixelated` | RESEARCH.md Pattern 3 |

**Note on "design system":** This is a Phaser game, not a web component app. All
interactive UI (menus, HUD, loading bar, toggles) is rendered as Phaser GameObjects
on a WebGL canvas. The design contract below specifies Phaser draw values (positions,
colors, text sizes in Phaser units = CSS px at 1x) and the thin HTML wrapper
(index.html body background, legal page styles). shadcn and Radix are out of scope
for all phases.

## Spacing Scale

All values are Phaser logical pixel units (= CSS px at 1x DPR). Must be multiples of 4.
Touch targets must be minimum 44px tall for WCAG mobile compliance.

| Token | Value | Usage in Phase 1 |
|-------|-------|------------------|
| xs | 4px | Icon gaps, inline text padding within Phaser Text objects |
| sm | 8px | Spacing between stacked menu items |
| md | 16px | Padding inside UI panels; progress bar vertical offset from text |
| lg | 24px | Section separation on MainMenu (title to CTA gap) |
| xl | 32px | Footer legal link strip height |
| 2xl | 48px | Vertical padding around MainMenu title block |
| 3xl | 64px | Top-of-screen safe margin (accounts for status bar + notch) |

Exceptions:
- Touch targets (audio toggle buttons, legal link hit areas): minimum 44px tall regardless of visual size, per WCAG 2.5.5. The visible sprite may be 16x16px; the Phaser input zone is 44x44px.
- Progress bar width: 300px (fixed, centered), height 12px with 2px stroke border at 14px. Source: RESEARCH.md Pattern 8.

## Typography

**Approach:** All in-game text uses Press Start 2P loaded as a Phaser BitmapFont or
Web Font. This is a fixed-width 8px-grid bitmap font — sizes must be multiples of 8px
for crisp rendering. Do not use fractional sizes. Web fallback for legal HTML pages:
`'Press Start 2P', monospace`.

| Role | Phaser size | Weight | Line Height | Usage |
|------|-------------|--------|-------------|-------|
| Display | 24px | 400 (bitmap — no bold variant needed) | 1.4 | Game title "BARREN WUFFETT" on loading/main menu |
| Heading | 16px | 400 | 1.4 | Section labels, menu option text, settings labels |
| Body | 8px | 400 | 1.5 | Legal footer links, progress label, version text |
| Label | 8px | 400 | 1.2 | Toggle state indicators ("ON" / "OFF"), short inline labels |

**Why only 2 sizes in some roles share 8px:** Press Start 2P's 8px grid means 8px is the
minimum readable size. Label and Body both use 8px but differ in line-height
application — Body is multi-word, Label is single short string.

**Legal HTML pages** (impressum.html, datenschutz.html): Use system font stack
`'Press Start 2P', 'Courier New', monospace` with 14px body text, 20px headings,
1.6 line-height. These are plain-text reading pages — readability beats style purity.

## Color

**Palette source:** Derived from the confirmed Phaser GameConfig `backgroundColor: '#1a1a2e'`
(RESEARCH.md Pattern 1) and PWA `theme_color: '#1a1a2e'`. Expanded to a 4-color retro palette
matching classic Game Boy / NES night-mode aesthetics — consistent with the 16-bit parody tone.

| Role | Hex | Phaser integer | Usage |
|------|-----|----------------|-------|
| Dominant (60%) | `#1a1a2e` | `0x1a1a2e` | Canvas background, body background, legal page background |
| Secondary (30%) | `#16213e` | `0x16213e` | UI panel backgrounds, progress bar track, settings card fill |
| Accent (10%) | `#00ff88` | `0x00ff88` | Reserved — see list below |
| Text / contrast | `#e8e8e8` | `0xe8e8e8` | All body and heading text on dark backgrounds |
| Destructive | n/a | n/a | No destructive actions in Phase 1 |

**Accent `#00ff88` is reserved exclusively for:**
1. Progress bar fill (the growing bar segment in PreloadScene)
2. Progress bar stroke border
3. "Tap to Start" text on MainMenuScene (primary CTA — only interactive element on screen)
4. Audio toggle indicator when state is ON (the lit pixel in the toggle sprite)

Accent must NOT be used for: decorative borders, background patterns, version text,
legal link text, or any secondary labels. One accent color, four reserved applications.

**Source:** `#00ff88` is explicitly called out in RESEARCH.md Pattern 8 progress bar code
(`0x00ff88`) and Pattern 2 CTA text. Extending it to toggle ON state is a default decision.

**Legal pages HTML color override:** Legal pages use CSS, not Phaser. Apply:
- Background: `#1a1a2e`
- Text: `#e8e8e8`
- Links: `#00ff88` (hover: `#00cc66`)
- This ensures visual continuity when the player taps the legal link and lands on plain HTML.

## Copywriting Contract

All strings are stored in `public/locales/en/common.json` and accessed via `t(key)`.
No English string may be hardcoded in TypeScript scene files.

| Element | Key | Copy | Source |
|---------|-----|------|--------|
| Primary CTA — main menu | `menu.tapToStart` | `TAP TO START` | RESEARCH.md Pattern 2 |
| Loading label | `loading.progress` | `LOADING...` | RESEARCH.md Pattern 4 JSON |
| Game title display | `game.title` | `BARREN WUFFETT` | LEGL-02 — parody name only |
| Game subtitle | `game.subtitle` | `THE VALUE INVESTING ADVENTURE` | default |
| Music toggle label | `audio.music` | `MUSIC` | RESEARCH.md Pattern 4 JSON |
| SFX toggle label | `audio.sfx` | `SOUND FX` | default (abbreviated to fit 8px grid) |
| Toggle ON state | `audio.on` | `ON` | default |
| Toggle OFF state | `audio.off` | `OFF` | default |
| Impressum link | `legal.impressum` | `IMPRESSUM` | LEGL-01 |
| Privacy policy link | `legal.datenschutz` | `PRIVACY POLICY` | LEGL-01 |
| Back to game link (legal pages) | `legal.backToGame` | `← BACK TO GAME` | RESEARCH.md Pattern 7 — used in HTML, not Phaser |
| Settings heading | `settings.title` | `SETTINGS` | default |
| Version label | `meta.version` | `v0.1.0` | default — updated at release |

**Empty state:** Phase 1 has no list views or data-driven content. No empty state copy needed.

**Error state:** Phase 1 has one potential error — asset load failure. Phaser's loader emits
`loaderror` events. Display inline below the progress bar:
- Key: `loading.error`
- Copy: `LOAD ERROR — PLEASE REFRESH`
- Render: 8px body text, color `#ff4444` (error red, used nowhere else in Phase 1)

**Destructive actions:** None in Phase 1. Audio toggles are non-destructive state changes.
No confirmation dialogs required.

**Legal page copy (static HTML — not in i18n JSON for Phase 1):**
- impressum.html H1: `Impressum`
- impressum.html body: German-language redirect text pointing to compoundingknowledge.com/impressum with Geltungssatz notice. See RESEARCH.md Pattern 7 for exact copy.
- datenschutz.html H1: `Datenschutzerklärung`
- datenschutz.html body: Game-specific privacy policy naming Cloudflare Pages (hosting), IndexedDB (future local save), Supabase (future cloud save — not yet active), Brevo (future email — not yet active). Written in German. See RESEARCH.md Pattern 7 for structural notes.

**Tone rule (from ENGAGEMENT.md):** All copy must pass the test "Would this mechanic
embarrass the creator if written up in a blog post?" No FOMO language, no urgency
framing, no dark patterns. "TAP TO START" is imperative but warm — it's an invitation,
not a demand. The subtitle frames the game as an "adventure," not a "game."

## Scene-by-Scene Interaction Contract

### BootScene (no visible UI)

- Reads `localStorage` for `bw_music_on` and `bw_sfx_on` audio toggle state
- Initializes AudioService with stored values
- Transitions immediately to PreloadScene
- No user interaction; no rendering beyond the canvas background color `#1a1a2e`

### PreloadScene (loading screen)

Layout (centered in 480x854 canvas):

```
Y=300  BARREN WUFFETT              [Display, 24px, #e8e8e8, centered at x=240]
Y=340  THE VALUE INVESTING ADVENTURE [Body, 8px, #e8e8e8, centered at x=240]
Y=420  [progress bar track]        [Rectangle, 302x14px, stroke 2px #e8e8e8, fill #16213e]
Y=420  [progress bar fill]         [Rectangle, grows 0→300px, 12px tall, fill #00ff88]
Y=444  LOADING...                  [Body, 8px, #e8e8e8, centered, updates to percentage]
```

- Progress bar fill width = `300 * progressValue` where `progressValue` is 0.0–1.0 from Phaser `load.on('progress', ...)`
- No interaction during loading; player cannot skip
- Transitions to MainMenuScene on `load.on('complete', ...)`

### MainMenuScene (interactive)

Layout (centered in 480x854 canvas):

```
Y=120  [Game logo / placeholder art]  [Image, max 320x160px, centered at x=240]
Y=300  BARREN WUFFETT                 [Display, 24px, #e8e8e8, centered]
Y=332  THE VALUE INVESTING ADVENTURE  [Body, 8px, #e8e8e8, centered]
Y=427  TAP TO START                   [Heading, 16px, #00ff88, centered — primary CTA]
Y=800  IMPRESSUM    PRIVACY POLICY    [Body, 8px, #e8e8e8, two links, footer strip]
```

- "TAP TO START" blinks at 1Hz (on 500ms / off 500ms) — standard retro menu convention
- `pointerdown` anywhere on canvas triggers `this.sound.unlock()` then transitions to SettingsScene (Phase 1) or OverworldScene (Phase 2+)
  - **Note:** In Phase 1, tapping "TAP TO START" transitions to the SettingsScene (audio toggles) since there is no gameplay scene yet. This tests the iOS audio unlock gate.
- IMPRESSUM and PRIVACY POLICY are Phaser Text objects with `setInteractive()` — hit area extended to 44px tall minimum
- Tapping IMPRESSUM opens `/impressum` via `window.open('/impressum', '_self')`
- Tapping PRIVACY POLICY opens `/datenschutz` via `window.open('/datenschutz', '_self')`
- Both links must also exist in the HTML DOM below the `#game-container` div as `<a>` tags (invisible, `visibility: hidden`) for legal crawler accessibility. See RESEARCH.md Pitfall 3.

### SettingsScene (stub — Phase 1 scope)

Layout (centered in 480x854 canvas):

```
Y=80   SETTINGS                    [Heading, 16px, #e8e8e8, centered]
Y=200  MUSIC          [toggle]     [Heading, 16px, #e8e8e8 label + 44x44px toggle button]
Y=260  SOUND FX       [toggle]     [Heading, 16px, #e8e8e8 label + 44x44px toggle button]
Y=700  IMPRESSUM    PRIVACY POLICY [Body, 8px, #e8e8e8, footer — identical to MainMenu]
Y=780  [Back button or tap-anywhere-back]
```

Toggle button states:
- ON: pixel block filled with `#00ff88`; label appended with ` ON` in `#00ff88`
- OFF: pixel block filled with `#16213e` (secondary, dim); label appended with ` OFF` in `#e8e8e8`
- Tap toggles state + calls `AudioService.setMusicEnabled()` / `AudioService.setSfxEnabled()`
- State persists immediately to localStorage on each toggle
- No confirmation required (non-destructive)

Back navigation: "TAP TO START" area or a ← back sprite returns to MainMenuScene.

### Static Legal Pages (HTML, outside Phaser)

impressum.html and datenschutz.html are plain HTML served by Cloudflare Pages.

CSS applied to both (inline style or shared `legal.css`):

body {

## Accessibility Baseline (Phase 1 minimum)

This is a canvas game; full WCAG compliance is not required for the canvas surface.
However, the following minimums apply to Phase 1:

| Rule | Requirement | How Met |
|------|-------------|---------|
| Touch target size | 44x44px minimum (WCAG 2.5.5) | All interactive Phaser elements use `setInteractive({ useHandCursor: true })` with a 44x44 input zone even if the sprite is smaller |
| Color contrast on legal pages | 4.5:1 minimum (WCAG 1.4.3) | `#e8e8e8` on `#1a1a2e` = 12.6:1 contrast ratio — passes AAA |
| Legal page HTML links | Visible in DOM for crawlers | `<a>` tags exist in HTML DOM below canvas (not only inside Phaser canvas) |
| iOS audio unlock | User gesture required before audio | "Tap to Start" gate — `this.sound.unlock()` called inside `pointerdown` handler |
| No pinch-zoom | Portrait layout stability | `user-scalable=no` + `touch-action: none` |
