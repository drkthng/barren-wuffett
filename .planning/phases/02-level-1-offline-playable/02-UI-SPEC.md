# Phase 2 — UI Design Contract

> Visual and interaction contract for Phase 2: Level 1 — Offline Playable.
> Generated by gsd-ui-researcher. Verified by gsd-ui-checker.
>
> **Scope:** OverworldScene HUD (virtual joystick, pause button, interaction prompt),
> UIScene overlay (dialogue box, pause menu, HUD elements),
> BossScene (Mr. Market greed/panic phases, patience affordance),
> PaperThrowScene HUD (score/progress, instructions),
> Level complete / boss defeat screen (Barren's Wisdom quote + share button + continue),
> Share card (1200×630 static render),
> Save/autosave indicator,
> New i18n keys for all of the above.
> All UI is rendered to a Phaser 4 canvas. No React, no shadcn — Phaser GameObjects only.

---

## Design System

| Property | Value | Source |
|----------|-------|--------|
| Tool | none | Phase 1 locked — Phaser 4 canvas, not a component library |
| Preset | not applicable | No shadcn; stack is Phaser 4 + Vite + TypeScript |
| Component library | Phaser 4 GameObjects (Text, Rectangle, Graphics, Image, Container) | Phase 1 UI-SPEC — inherited |
| Icon library | None — pixel-art sprites only; UI icons hand-drawn in 16x16 tileset | Phase 1 UI-SPEC — inherited |
| Font | Press Start 2P (Google Fonts) — loaded as Phaser WebFont; sizes must be multiples of 8px | Phase 1 UI-SPEC — inherited |
| CSS scope | Only the HTML shell; Phaser canvas uses `image-rendering: pixelated` | Phase 1 UI-SPEC — inherited |

**Note on design system continuity:** Phase 2 inherits 100% of the Phase 1 design system.
No new fonts, no new base palette, no new grid system. Extensions in this phase are:
(a) two semantic boss-phase tints, (b) a semi-transparent overlay color, and
(c) a save indicator animation spec. All additions are Phaser draw operations on the
existing palette.

## Spacing Scale

All values are Phaser logical pixel units (= CSS px at 1x DPR). Inherited from Phase 1.
New Phase 2 usages noted in the Usage column.

| Token | Value | Phase 1 Usage | Phase 2 Additional Usage |
|-------|-------|---------------|--------------------------|
| xs | 4px | Icon gaps, inline text padding | Dialogue box inner text padding; boss health bar track border |
| sm | 8px | Spacing between stacked menu items | Dialogue portrait-to-text gap; pause menu item gap; boss phase label gap |
| md | 16px | Padding inside UI panels | HUD element margin from canvas edge; dialogue box horizontal padding |
| lg | 24px | Section separation on MainMenu | Pause menu item height (touch target area); level complete quote block padding |
| xl | 32px | Footer legal link strip height | Dialogue box total height (portrait slot 32px tall); share card bottom strip height |
| 2xl | 48px | Vertical padding around MainMenu title | Level complete screen section spacing; boss health bar Y position from top |
| 3xl | 64px | Top-of-screen safe margin | HUD top margin (status bar + camera offset); dialogue box Y anchor from bottom |

Phase 2 Exceptions:
- Virtual joystick base: 80px radius circle. Not on the 8-point scale — this is a Phaser VirtualJoystick `radius` parameter, not a layout spacing token. Source: RESEARCH.md Pattern 2.
- Virtual joystick position: `x=120, y=750` (portrait canvas). Kept at these values to respect the iOS safe-area-inset-bottom. Source: RESEARCH.md Pitfall 6.
- Boss phase indicator tint: fills 100% canvas width as a background overlay. Not a spacing token.
- Save indicator icon: 16x16px sprite (smallest tileset unit). Not a spacing token.
- Dialogue box height: 160px (5 lines of 8px text at 1.5 line-height with 16px padding top/bottom). Value is derived, not a standalone token.

## Typography

Inherited from Phase 1. Press Start 2P fixed-width bitmap font. All sizes multiples of 8px.
No new sizes introduced in Phase 2 — the four Phase 1 roles cover all new UI surfaces.

| Role | Phaser size | Weight | Line Height | Phase 2 Usage |
|------|-------------|--------|-------------|---------------|
| Display | 24px | 400 | 1.4 | Level complete heading ("LEVEL COMPLETE"); boss defeat quote attribution |
| Heading | 16px | 400 | 1.4 | Pause menu items; boss phase label ("GREED PHASE" / "PANIC PHASE"); dialogue NPC name; mini-game score; share button label |
| Body | 8px | 400 | 1.5 | Dialogue text (typewriter renders at 8px, word-wrapped via Phaser Text `wordWrap`); HUD coin counter; interaction prompt ("TALK"); autosave indicator label; level complete quote body; boss decline/accept button labels |
| Label | 8px | 400 | 1.2 | Boss health tick marks; mini-game timer countdown; patience bonus indicator ("PATIENCE +5"); share card URL text |

**Dialogue box font size decision (AUTO MODE):** RESEARCH.md Assumption A6 flagged that
8px might be too small for dialogue on real devices. This spec defaults to **8px** for
dialogue body text to maintain grid consistency, but the dialogue box must be tested on
real iPhone SE before shipping. If readability fails, the fallback is 16px Body —
the font size line items above already define 16px as the Heading role, so no new token
is needed for that fallback. This is a revisable default.

**Share card typography:** The share card is rendered on an offscreen 1200×630 canvas
using native 2D canvas API (not Phaser). Sizes on the share card are scaled up 2.5x
from the game canvas:
- Quote text: `60px "Press Start 2P"` (equivalent to Display 24px × 2.5)
- Game URL: `40px "Press Start 2P"` (equivalent to Heading 16px × 2.5)

**Phase 2 Color Extensions:**

| Role | Hex | Phaser integer | Usage | Source |
|------|-----|----------------|-------|--------|
| Boss greed tint | `#ff4444` | `0xff4444` | Full-canvas background tint overlay (alpha 0.15) during Mr. Market Greed Phase — signals danger/overvaluation | RESEARCH.md Game Design: Boss Fight |

| Overlay backdrop | `#000000` | `0x000000` | Semi-transparent black at alpha 0.6 for pause menu backdrop and dialogue box backdrop | default |

**Accent `#00ff88` is reserved exclusively for:**
1. Progress bar fill (Phase 1 — inherited)
2. Progress bar stroke border (Phase 1 — inherited)
3. "TAP TO START" CTA text (Phase 1 — inherited)
4. Audio toggle ON indicator (Phase 1 — inherited)
5. Pause button icon sprite (Phase 2 — the only active HUD button in overworld)
6. Interaction prompt text "TALK" near NPCs (Phase 2 — signals available action)
7. Patience bonus flash text (Phase 2 — "+5 PATIENCE BONUS" popup, 1-second flash)
8. Share button highlight on level complete screen (Phase 2 — primary CTA on that screen)
9. Boss defeat "BUY LOW!" text flash (Phase 2 — the win-state moment label)
10. Autosave indicator disk icon (Phase 2 — appears as a 16x16px disk sprite in accent color)

Accent must NOT be used for: decorative tilemap borders, NPC name labels, dialogue
advance indicators, boss health bars, mini-game score text, level complete quote body
text, or any secondary informational labels.

**Boss tint implementation note:** The greed/panic tints are applied as a
`Phaser.GameObjects.Rectangle` covering the full canvas (480×854) at the specified
alpha — not as a camera color matrix filter. This is simpler and performs well on mobile.
The rectangle sits in the BossScene at a depth below sprites but above the tilemap layer.

## Component Inventory

### 1. Overworld HUD (rendered in UIScene at fixed camera depth)

**Virtual Joystick**
- Library: `phaser4-rex-plugins` VirtualJoystick — already installed
- Base position: `x=120, y=750` (portrait canvas 480×854)
- Base radius: 80px; fill color `#16213e`; stroke `#e8e8e8` at alpha 0.6, width 2px
- Thumb radius: 32px; fill color `#e8e8e8` at alpha 0.8
- Direction: `'8dir'`; forceMin: 16
- Visible only when: `this.sys.game.device.input.touch === true` (hidden on desktop)
- Z-depth: rendered above tilemap, above player sprite, below dialogue box

**Pause Button**
- Position: `x=448, y=32` (top-right, 32px from right edge, 32px from top — inside 3xl safe margin)
- Size: 32x32px sprite; input zone extended to 44x44px (WCAG)
- Visual: pixel-art pause icon (two vertical bars, 8px each, spaced 4px apart) in `#00ff88`
- On press: emits `GameEvents.PAUSE_REQUESTED`; button depresses 4px (alpha 0.7 for 100ms feedback)
- Always visible in overworld; hidden in dialogue state

**Interaction Prompt ("TALK")**
- Appears when player sprite is within 48px of an NPC interaction zone
- Position: above the NPC sprite, offset `y = npc.y - 40`; centered on NPC x
- Text: `t('hud.talkPrompt')` → `TALK` in Body 8px, color `#00ff88`
- Blink: 1Hz (500ms on / 500ms off) — same rhythm as "TAP TO START" (Phase 1 convention)
- Disappears when player moves outside 48px radius or dialogue triggers

**Coin Counter**
- Position: `x=16, y=16` (top-left corner)
- Sprite: 16x16px pixel-art coin icon + Body 8px text `t('hud.coins')` format: `COINS: {n}`
- Text color: `#e8e8e8`
- Updates on coin pickup event; brief scale-up tween (1.0 → 1.2 → 1.0 over 150ms) on increment

**Autosave Indicator**
- Position: `x=16, y=838` (bottom-left, 16px from bottom)
- Visual: 16x16px disk icon sprite in `#00ff88`
- Behavior: invisible at rest; fades in (alpha 0 → 1 over 200ms) then fades out (alpha 1 → 0 over 600ms) on SaveService.save() call
- No text label — icon-only to minimize screen real estate
- Total duration of flash: 800ms

### 2. Dialogue Box (rendered in UIScene)

**Panel geometry**
- Position: anchored to bottom of canvas; `x=0, y=694` (854 - 160px height)
- Size: 480px wide × 160px tall
- Fill: `#1a1a2e` at alpha 0.95; stroke: `#16213e` 2px top border only
- Backdrop: full-canvas `#000000` Rectangle at alpha 0.3, Z-depth below dialogue panel

**Portrait slot**
- Position: `x=16, y=710` (16px from left, 16px below panel top)
- Size: 48x48px
- Content: NPC portrait sprite keyed from dialogue JSON `portrait` field
- If portrait key not found: render `#16213e` filled square as placeholder (no error thrown)
- Border: 2px `#e8e8e8` rectangle stroke around portrait

**NPC Name Label**
- Position: `x=76, y=710` (4px gap after portrait at x=64+4=68... adjusted to x=76 for 8px grid alignment)
- Font: Heading 16px, color `#e8e8e8`
- Content: `t('npc.{npcId}.name')` — pulled from i18n, not embedded in dialogue JSON
- Max width: 388px (480 - 76 - 16 padding)

**Dialogue Text**
- Position: `x=76, y=734` (below name label with 8px gap: 710 + 16 + 8 = 734)
- Font: Body 8px, color `#e8e8e8`
- Word wrap: `{ width: 388, useAdvancedWrap: true }` on Phaser Text object
- Max visible lines: 4 lines at 8px × 1.5 line-height = 48px text block
- Content: renders via typewriter at 30ms per character (Phaser `time.addEvent`)
- Skip/advance: `pointerdown` anywhere on dialogue panel skips typewriter if running, or advances to next line if typewriter complete

**Advance Indicator**
- Position: `x=456, y=830` (bottom-right of panel, 8px from right edge, 8px from bottom)
- Visual: `▼` character at Body 8px, color `#e8e8e8`
- Blink: 1Hz — only shown when typewriter is complete and more lines remain
- Hidden when on last line of dialogue (replaced by nothing — tap advances to close dialogue)

**Dialogue close behavior:** On last line, `pointerdown` closes dialogue, hides panel,
resumes OverworldScene, emits `GameEvents.DIALOGUE_COMPLETE` with NPC id.

### 4. Boss Scene UI — Mr. Market (BossScene overlay, rendered in BossScene)

**Canvas tint overlay**
- Greed Phase: `#ff4444` Rectangle at alpha 0.15, full canvas, Z-depth 0 (behind all sprites)
- Panic Phase: `#4444ff` Rectangle at alpha 0.15, full canvas
- Transition: tween alpha 0→0.15 over 500ms when phase changes

**Boss Health / State Indicator**
- Position: `x=240, y=48` (top-center)
- Label: `t('boss.mrMarket.name')` → `MR. MARKET` in Heading 16px, color `#e8e8e8`, centered
- Phase label: `t('boss.phase.greed')` → `GREED PHASE` or `t('boss.phase.panic')` → `PANIC PHASE`
  - Font: Body 8px, centered below name, color `#ff4444` (greed) / `#4444ff` (panic)
  - Transitions: text swaps when phase changes; scale-up tween (1.0 → 1.3 → 1.0 over 300ms)
- State bar: 5 tick marks (5 "offers" in Greed Phase) rendered as 16x12px `#16213e` Rectangles
  in a row, centered at `x=240, y=80`, gap 4px between marks
  - Remaining ticks: fill `#ff4444` (greed) or `#4444ff` (panic); depleted ticks: fill `#16213e`

**Patience Mechanic Affordance (GAME-06 — how the player SEES that waiting is rewarded)**

This is the critical interaction contract for the boss fight. The player must understand
that DECLINING in Greed Phase is the winning move, and that WAITING through all 3 declines
produces a bonus.

Implementation:
- Greed Phase: each DECLINE increments a "Patience Meter" shown as a small bar
  below the state ticks. Label: `t('boss.patienceMeter')` → `PATIENCE:` at Body 8px.
  Bar: 3 segments (one fills per DECLINE), fill color `#00ff88`, track color `#16213e`.
  Width: 48px (3 × 16px segments); position `x=216, y=96` (bar starts left of center)
- Greed Phase text hint (shown only on first encounter, dismissed after first DECLINE):
  `t('boss.hint.decline')` → `DECLINING BUILDS PATIENCE!`
  Font: Body 8px, color `#00ff88`, centered at `x=240, y=120`
  Auto-dismisses after 3 seconds or on first pointerdown
- After all 3 Greed Phase declines, flash: `t('boss.patienceFull')` → `PATIENCE MAXED!`
  in `#00ff88` at Heading 16px for 1 second, then Panic Phase begins

**DECLINE Button (Greed Phase)**
- Position: `x=240, y=700`
- Size: 240px wide × 56px tall; fill `#16213e`; stroke `#00ff88` 2px
- Label: `t('boss.decline')` → `DECLINE` at Heading 16px, color `#00ff88`
- On press: depletes one greed tick, increments patience meter, advances to next "offer" line
- Feedback: button scale-down (1.0 → 0.95 → 1.0, 100ms); coin counter shakes (reject animation)

**ACCEPT Button (Panic Phase)**
- Position: `x=240, y=700`
- Size: 240px wide × 56px tall; fill `#16213e`; stroke `#4444ff` 2px (panic color)
- Label: `t('boss.accept')` → `BUY LOW!` at Heading 16px, color `#00ff88`
  (Note: label says "BUY LOW!" not "ACCEPT" — frames the action as the investing lesson)
- Countdown timer: `t('boss.timer')` → `{n}s` at Body 8px, color `#ff4444`
  Position: centered above button at `x=240, y=680`; counts down 3→0
  Timer starts when Panic Phase begins. On expiry: GAME-OVER-like state
  (see "Patience miss outcome" below)
- On press in time: emits `GameEvents.BOSS_DEFEATED`; triggers level complete sequence

**Patience miss outcome (timer expires without pressing BUY LOW!):**
- Screen flash `#ff4444` alpha 0.4 for 200ms
- Text: `t('boss.tooSlow')` → `TOO SLOW! THE MARKET RECOVERED!`
  Font: Heading 16px, color `#ff4444`, centered at canvas center, duration 2 seconds
- Then: Greed Phase resets (Mr. Market returns; state ticks reset). Player can try again.
  No permanent fail state — retry immediately without energy cost (per ENGAGEMENT.md anti-pattern 1)

**Boss offer text (displayed in a speech bubble above Mr. Market sprite)**
- Speech bubble: `#16213e` rounded Rectangle (cornerRadius 8px) at `x=240, y=320`
  width 280px, height 48px
- Text: dialogue pulled from `data/dialogue/en/level-01.json` boss offer lines
  Font: Body 8px, `#e8e8e8`, centered in bubble

## Scene-by-Scene Interaction Contract

### OverworldScene

**Input sources:**
- Virtual joystick (touch) → InputBus.activeActions (MOVE_UP/DOWN/LEFT/RIGHT)
- Keyboard arrows/WASD → InputBus.activeActions
- Pause button tap → `GameEvents.PAUSE_REQUESTED`
- INTERACT action → `GameEvents.DIALOGUE_START` with NPC id (when player in NPC zone + INTERACT)
- INTERACT on desktop: Spacebar or Enter key → INTERACT action in InputBus

**Movement feedback:**
- Player sprite: 4-directional walk animation (2-frame minimum; 4-frame preferred if assets allow)
- Walking on floor tiles: no SFX in Phase 2 (audio content deferred — mechanics first)
- Entering boss trigger zone: brief screen flash `#4444ff` alpha 0.1 for 300ms (foreshadowing)

**Dog patrol patience mechanic (GAME-06):**
- Dog sprite patrols 80px range, period 4 seconds (2s each direction)
- Gap window: 1.5 seconds (when dog is at far end of patrol, route is clear)
- Visual: dog sprite moves on path; no UI affordance needed — player learns by observing
- Patience bonus: awarded when player uses the gap route; `GameEvents.PATIENCE_BONUS` emitted
  with value +5; HUD flashes `+5 PATIENCE BONUS` at Body 8px, `#00ff88`, centered at `x=240, y=200`
  for 1 second, floats up (tween y 200→160, 800ms)

**Autosave trigger points:** NPC dialogue complete; mini-game complete; boss defeated;
entering boss zone for first time. Each trigger calls `SaveService.save()` → triggers
the autosave disk flash.

### UIScene

Parallel scene running above OverworldScene, BossScene, and PaperThrowScene.
Owns: HUD container, DialogueBox container, PauseMenu container, LevelCompleteOverlay container.
Only one container is visible at a time (managed by UIScene state machine).

UIScene states:
1. `HUD` — default; shows coin counter, pause button, autosave indicator
2. `DIALOGUE` — shows dialogue box + backdrop; HUD hidden; input captured by dialogue box
3. `PAUSE` — shows pause overlay; all game scenes paused
4. `LEVEL_COMPLETE` — shows level complete overlay; all game scenes stopped

Transitions:
- HUD → DIALOGUE: on `GameEvents.DIALOGUE_START`
- DIALOGUE → HUD: on `GameEvents.DIALOGUE_COMPLETE`
- HUD → PAUSE: on `GameEvents.PAUSE_REQUESTED`
- PAUSE → HUD: on `GameEvents.RESUME_REQUESTED`
- Any → LEVEL_COMPLETE: on `GameEvents.BOSS_DEFEATED`

### BossScene

Separate scene (not overlay). OverworldScene sleeps while BossScene runs.

**Phase transitions:**
1. Scene starts → 1-second camera pan to Mr. Market sprite → phase label "GREED PHASE" appears
2. Greed Phase: 3 "offer" lines shown sequentially in speech bubble; player taps DECLINE each time
3. After 3rd DECLINE: "PATIENCE MAXED!" flash → tint transitions from `#ff4444` to `#4444ff` over 500ms
4. Panic Phase: 3-second countdown timer visible; ACCEPT button ("BUY LOW!") appears
5. Player taps BUY LOW! in time → "BUY LOW!" flash in `#00ff88` at Display 24px for 1s
6. BossScene emits `GameEvents.BOSS_DEFEATED`; ShareService.shareCard pre-generates Blob
7. BossScene stops; OverworldScene wakes; UIScene transitions to LEVEL_COMPLETE state

### PaperThrowScene

Launched from OverworldScene (overworld sleeps). Mini-game runs in its own scene.

**HUD ownership:** PaperThrowScene renders its own score/timer HUD (not UIScene)
because UIScene's HUD coin counter should not be visible during the mini-game —
the mini-game has its own scoring context. On mini-game complete, score converts
to coins and UIScene HUD is updated.
