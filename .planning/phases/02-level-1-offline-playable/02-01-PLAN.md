<objective>
Build the playable Level 1 overworld vertical slice: a real tilemap the player walks through with joystick + keyboard, NPCs they talk to via a typewriter dialogue box, a pause/resume overlay, a patience-rewarding dog-patrol mechanic, and IndexedDB save that survives refresh. This is the MVP happy path — after this plan a player can boot the game, walk Omaha, talk to Grandpa, pause, and have their position persist across a reload, fully offline.

This plan also lays the shared seams (GameEvents bus, fully-wired InputBus, SaveService, level/dialogue data, all scene registrations, all i18n keys, all asset preloads) that Plans 02 and 03 consume without re-touching main.ts, Preloader.ts, or common.json — mirroring the Phase 1 atomic-registration pattern that kept later plans conflict-free.

Purpose: Prove the core loop (move → interact → save) works end-to-end on a real device before investing in boss/share systems.
Output: Walkable, talkable, pausable, persistent Level 1 overworld + Wave 0 test scaffold.
</objective>

**As a** player, **I want to** walk through Level 1 Omaha with a joystick or keyboard, talk to NPCs, pause anytime, and have my progress saved locally, **so that** I can play the start of the game on my phone with no account and pick up where I left off after a refresh.
